# 第三轮迭代记录

日期：2026-06-16

## 目标

围绕长文本体验、传输体积、播放反馈、托盘自启和边界测试，完成专家组 P1
中投入产出比最高的改进。

## 已完成

- 新增 `audio_encoding.py`，通过 `imageio-ffmpeg` 提供 bundled FFmpeg。
- `/tts` 保持默认 WAV，同时支持 `Accept: audio/ogg` 与 `?format=ogg` 输出 OGG/Opus。
- 新增 `/tts/stream`，每个流请求启动独立 FFmpeg 子进程，将 Kokoro 分段 PCM 写入 stdin，并从 stdout 连续返回 WebM/Opus。
- 用户脚本优先使用 MediaSource + WebM/Opus 流式播放；不支持时回退到 OGG/Opus。
- 浮动按钮改为横向填充进度：流式缓冲阶段显示已播放秒数，流结束且总时长可用后显示百分比。
- 托盘菜单新增 `Auto-start on login`，通过 Windows Startup 快捷方式启动 `Kokoro TTS.pyw`。
- CI 切换为 `pytest`，并验证 bundled FFmpeg 可执行。
- 增加边界与回归覆盖：纯标点、数字、URL、中英文混合、并发锁、OGG 输出、WebM 首包、分段静音、托盘自启。

## API 变化

- `POST /tts`
  - 默认：`audio/wav`
  - `Accept: audio/ogg` 或 `?format=ogg`：`audio/ogg`
- `POST /tts/stream`
  - 返回：`audio/webm; codecs="opus"`
  - 用于浏览器 MediaSource 连续播放
