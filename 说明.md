# 本地划词听译助手

> 在 Chrome 浏览器中选中网页文本，一键本地朗读或本地翻译。
> 朗读由 **Kokoro TTS** 完成，翻译由 **Ollama** 本地模型完成，文本不上传云端。
> `Read` 会先清洗文本：英文公式场景会先读正文、后台处理公式，播放到公式时如果还没处理好再等待；中文、代码块、URL、表格碎片等不会直接送进英文 TTS。

## 为什么选 Kokoro TTS？

| 特性 | 说明 |
|------|------|
| 🏆 发音质量 | 开源本地 TTS 中最自然标准的英文发音，被称为"TTS 界的 LLaMA 时刻" |
| ⚡ 极速推理 | 82M 超轻量模型，RTX 4070 Super 上几乎实时生成 |
| 🌊 流式播放 | Chrome 使用 `/tts/stream` + MediaSource + WebM/Opus，长文本可边生成边播放 |
| 📦 压缩输出 | `/tts` 支持 OGG/Opus，默认 WAV 保持兼容 |
| 🛡️ 智能调度 | 后端支持智能排队机制与客户端断开感知，防止并发导致显卡 OOM 及僵尸进程 |
| 🧹 优雅清理 | 前端内置 MutationObserver 与 AbortController，页面路由跳转或切换时自动清理内存与请求 |
| 📊 进度反馈 | 浮动按钮显示横向填充进度；流式阶段先显示已播放秒数 |
| 🧠 渐进朗读稿 | 英文含公式时先读正文，后台用本地模型处理公式；到公式处若尚未完成再等待，避免整段被公式拖住 |
| ∑ 公式友好 | MathJax/MathML/LaTeX 会优先提取语义公式；朗读时转成流畅口语，翻译时在前端渲染为带上下标的易读公式 |
| 🧭 上下文参考 | 翻译请求可附带附近正文作为本地参考，只用于术语和指代消歧；真正翻译和输出的只有选中内容 |
| 🧮 模型上下文预算 | 4B 模型翻译和公式朗读不参考上下文，9B/14B/更大模型按大小逐级增加上下文长度 |
| ⚡ Qwen3 no-think | `qwen3:14b`、QwQ、DeepSeek-R1 等推理模型会通过 Ollama `think: false` 关闭思考过程，减少翻译和朗读准备延迟 |
| ∑ 4B 保守公式读法 | 选择 4B 模型时，常见公式优先用本地字面规则朗读，例如 `D_I` 读作 `D sub I`，`\hat{B}(x)` 读作 `B hat of x` |
| ∑ 数学术语表 | `config/math_glossary.json` 维护 50+ 个核心数学符号的直接读法和上下文读法，例如右箭头可按语境读作映射到、趋向于、推导出、得到或右箭头 |
| 🧭 避让选区 | Read 和 Translate 可同时进行；按钮行固定在选区下方，译文卡片会根据空间避让，减少遮挡正文 |
| ∑ 局部公式恢复 | 只选中 MathJax/MathML/KaTeX 公式的一部分时，会尽量扩展到完整公式框；选中整句时保留公式前后文字 |
| 🎭 17 种声音 | 内置美式男声、女声、英式声音，无需自备参考音频 |
| 🔒 隐私安全 | 完全本地运行，不发送任何数据到外部服务器 |
| 💰 免费无限 | 开源免费，无 API 调用费用 |

## 🚀 快速开始（5 步走）

### 第 1 步：安装 eSpeak-NG

Kokoro 依赖 eSpeak-NG 进行音素转换，**必须先安装**：

1. 访问 [eSpeak-NG Releases](https://github.com/espeak-ng/espeak-ng/releases)
2. 下载最新的 Windows `.msi` 安装包并安装
3. **关键**：将 eSpeak-NG 安装路径（通常是 `C:\Program Files\eSpeak NG`）添加到系统环境变量 `Path` 中
4. 重启终端使环境变量生效

验证安装：
```powershell
espeak-ng --version
```

### 第 2 步：创建 Conda 环境 & 安装依赖

```powershell
# 创建独立的 Python 3.10 环境
conda create -n kokoro-tts python=3.10 -y
conda activate kokoro-tts

# 安装 PyTorch（CUDA 12.4，兼容你的 CUDA 13.1 驱动）
pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu124

# 安装项目依赖
pip install -r requirements.txt
```

或者直接双击 **`setup.bat`** 一键完成。

### 第 3 步：启动 TTS 服务器

```powershell
conda activate kokoro-tts
python server.py
```

首次启动会自动从 HuggingFace 下载 Kokoro 模型（约 200MB，非常小），之后启动只需几秒。
看到 `✅ 模型加载完成！服务已就绪。` 表示服务可用。

也可以直接双击 **`start.bat`** 一键启动。

推荐双击 **`Kokoro TTS.bat`** 启动托盘程序；它不依赖 Windows 的 `.pyw` 文件关联。
`start.bat` 和 `Kokoro TTS.bat` 会直接定位 `kokoro-tts` 环境里的 Python，日常启动不需要先执行 `conda init`。`Kokoro TTS.pyw` 保留为无黑窗口启动器，但只有在 Windows 已把 `.pyw` 关联到 Python 时才适合双击。

### 第 4 步：安装油猴脚本

1. 在 Chrome 中安装 [Tampermonkey 扩展](https://chrome.google.com/webstore/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo)
2. 推荐从 Greasy Fork 安装正式发布版
3. 如需开发版，可打开 GitHub raw 脚本：`https://raw.githubusercontent.com/Yan-ShiBo/LocalReadTranslate/main/tts-userscript.js`
4. Tampermonkey 弹出安装页后确认安装

### 第 5 步：开始使用！

1. 确保 TTS 服务器正在运行（第 3 步）
2. 打开任意英文网页
3. **选中一段网页文本** → 自动弹出 `Read` 和 `Translate` 按钮
4. 点击 `Read` 本地朗读英文内容并后台处理公式，或点击 `Translate` 使用本机 Ollama 模型翻译 🎉

> ⌨️ **快捷键**：选中文本后按 **Ctrl+Shift+S** 也可直接朗读。
> 翻译、朗读稿准备和复杂公式默认都使用 `translategemma:4b`；可通过 `OLLAMA_TRANSLATE_MODEL`、`OLLAMA_READ_MODEL`、`OLLAMA_FORMULA_MODEL` 覆盖。
> 上下文长度会按模型大小自动裁剪：4B 翻译和公式朗读不参考上下文，14B 会保留更多上下文，避免小模型被长上下文干扰。
> 使用 `qwen3:14b`、QwQ、DeepSeek-R1 等推理模型时，服务端会自动向 Ollama 传入 `think: false`，让翻译和朗读准备直接输出结果。
> 4B 公式朗读会尽量不用上下文判断语义，而是采用最常见、最保守的字面读法；规则处理不了的复杂公式才交给 Ollama。
> 公式口语化会参考 `config/math_glossary.json`，先用规则处理常见符号，再把同一术语表交给 Ollama 处理复杂公式；当前表吸收了 v2 参考表，覆盖箭头、上下标、集合、逻辑、求和、积分、偏导等核心符号。
> 如果朗读稿准备接口不可用，`Read` 会退回到 `/translate` 并指定翻译成 English 后再朗读。

## 🔌 API 接口说明

### `POST /tts` — 文本转语音

**请求：**
```json
{
  "text": "Hello, how are you today?",
  "voice": "af_bella",
  "speed": 0.8
}
```
`voice` 可选，默认使用 `af_bella`。`speed` 可选，默认 `0.8`。

**响应：** 默认 `audio/wav`；使用请求头 `Accept: audio/ogg` 或 `?format=ogg` 可返回 OGG/Opus。

**示例（PowerShell）：**
```powershell
Invoke-WebRequest -Uri "http://127.0.0.1:5000/tts" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{"text":"Hello world"}' `
  -OutFile "test.wav"
```

### `POST /tts/stream` — 流式文本转语音

返回 `audio/webm; codecs="opus"`，用于油猴脚本的 MediaSource 连续播放。长文本会在全部生成完成前开始播放。

### `GET /health` — 健康检查
### `GET /` — 内置测试页面（浏览器访问 http://127.0.0.1:5000）

## ❓ 常见问题

### Q：和 Ollama 有什么关系？
A：当前版本里，Ollama 用于**本地翻译**，Kokoro TTS 用于**本地语音朗读**。两者都由本地 FastAPI 服务调度，浏览器脚本只请求 `127.0.0.1:5000`。

### Q：Greasy Fork 发布页的 GitHub 地址放哪里？
A：脚本元信息里使用 `@homepageURL` 指向 GitHub 项目主页，`@supportURL` 指向 GitHub Issues；发布页“附加信息”正文里也建议再放一次项目地址。可直接使用 `docs/greasyfork-additional-info.md` 的内容。

### Q：推理速度如何？
A：RTX 4070 Super 上，一句普通英文几乎瞬间生成（<0.5 秒）。Kokoro 只有 82M 参数，极其轻量。

### Q：如何更换声音？
A：修改 `server.py` 顶部的 `VOICE` 变量，或在 API 请求中指定 `"voice": "am_liam"` 等。

### Q：eSpeak-NG 安装后仍报错？
A：确认已将 `C:\Program Files\eSpeak NG` 添加到系统 Path 环境变量，且**重启了终端**。

### Q：油猴脚本不工作？
A：确认服务器正在运行（浏览器访问 `http://127.0.0.1:5000/health`），以及 Tampermonkey 脚本已启用。

## ⚙️ 高级配置

在 `server.py` 顶部可修改以下配置：

```python
HOST = "127.0.0.1"       # 监听地址
PORT = 5000               # 监听端口
VOICE = "af_bella"         # 默认声音（甜美阳光女声）
DEVICE = "auto"            # auto / cuda / cpu
# speed 默认 0.8（稍慢于正常语速，更自然连贯）
```

在 `tts-userscript.js` 顶部可修改：

```javascript
const API_URL = 'http://127.0.0.1:5000/tts';  // API 地址
```
